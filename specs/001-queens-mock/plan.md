# Implementation Plan: Queems Queens Puzzle Mock

**Branch**: `001-queens-mock` | **Date**: 2026-02-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-queens-mock/spec.md`

## Summary

Build a browser-based mock of the LinkedIn Queens puzzle game. Players can revisit all available
stages from a stage-selection screen and set personal speed records (fastest completion time per
stage) that persist in browser storage. Stack: Vite SPA + React 19 + TypeScript 5 + Tailwind
CSS v4 + Zustand 5 + React Router v7 + Framer Motion v11 + Lucide React.

## Technical Context

**Language/Version**: TypeScript 5 (`"strict": true`)

**Scale/Scope**: ≥ 5 pre-defined stages; single-player; single-device persistence

## Phase 1: Design — COMPLETE

### Architecture Decisions

1. **Pure function isolation**: `rule-validator.ts` and `board-state.ts` are stateless pure
   functions — no imports of React, no Zustand. They accept arguments and return values.
   This makes TDD trivial and Constitution II/IV fully satisfied.

2. **Two Zustand stores** (ephemeral + persisted): `game-store` resets when a stage is loaded
   or restarted. `best-times-store` is independent and never cleared during gameplay — only
   `saveBestTime()` mutates it.

3. **ConflictMap is always derived, never persisted**: On every `placeOrRemoveQueen()` action
   in `game-store`, the component calling the store re-computes `deriveConflicts(queens, stage)`
   before rendering. Given ≤ N² cells (N ≤ 10), this is O(N²) and completes in < 1ms.

4. **Timer lifecycle in PuzzlePage**: `useEffect` in `PuzzlePage` sets up a `setInterval`
   calling `game-store.tick()` every second. The cleanup function clears the interval.
   The interval only runs when `!isSolved`.

5. **Region colors via lookup table**: `src/lib/region-colors.ts` maps `RegionId` strings to
   Tailwind CSS class names. Cell component uses `cn()` to merge region color with state classes.

### Data Model Summary (see `data-model.md`)

| Stage | Static import | Immutable |
| Queen[] | game-store (Zustand) | Ephemeral |
